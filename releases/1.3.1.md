# Release v1.3.1 - 2026-03-02

## Highlights
- Improved update-check UX on Home with clearer, state-based button feedback.
- Added app version display on Home screen.
- Expanded test coverage for core support-search behavior and update UI flows.

## Changes
- Replaced update toast messaging with inline update status directly on the update button.
- Added loading spinner and polling flow while checking for updates.
- Added `Current version` label in Home footer.
- Added stable `data-testid` hooks on navigation and support-search screens for E2E reliability.
- Refactored Playwright E2E scenarios to cover:
  - Home render and update button visibility
  - City match search with route request
  - Fallback to nearest assistance when city has no match (without route request)
- Added unit tests for assistance selection rules:
  - nearest assistance by city
  - global nearest fallback
- Increased Playwright timeout for better CI stability.
- Kept update checks disabled for E2E runtime (`CI=e2e`) to avoid flaky external calls.

## Validation Checklist
- [ ] Home screen shows current app version.
- [ ] Update button shows spinner and status transitions.
- [ ] Search with city match draws route and displays nearest in-city assistance.
- [ ] Search without city match shows nearest fallback and no route request.
- [ ] Unit tests pass in CI.
